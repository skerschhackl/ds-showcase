# Design System Principles

This design system is a governed interface language for humans and AI agents.

Its visual direction is a playful creative operating system: soft, spacious, tactile, and friendly, while still governed enough for real product UI.

For enforceable AI-facing principles, use `specs/foundation/principles.md`.

## Human Principles

1. Shared language beats isolated assets.
2. Components should encode product behavior, not just appearance.
3. Tokens are the source of visual truth.
4. AI output must be reviewable against explicit rules.
5. New patterns earn their way into the system through repeated need.

## How To Use This

Use these principles when making product and governance decisions. Use the `specs/` folder when an AI agent needs operational instructions.
